# ADR-12 — Eval metric suite + eval pin path

- **Tarih:** 2026-05-28
- **Durum:** Kabul edildi
- **Bağlam:** ADR-7 `voices.eval_metrics` JSONB kolonunu koydu (voice manifest schema v2 forward-shape). ADR-8 LoRA fine-tune pipeline'ını çıkardı; eval ile entegrasyonu eksik. Mevcut `src/eval/`:
  - **Whisper-WER + Whisper-CER** ([src/eval/metrics/whisper_wer.py](../../src/eval/metrics/whisper_wer.py)) — production-quality, large-v3, Türkçe transcript cleanup, model-sharing pattern.
  - **UTMOSv2** ([src/eval/metrics/utmosv2.py](../../src/eval/metrics/utmosv2.py)) — protokol iskelesi mevcut, `_load()` `NotImplementedError` raise ediyor ("PR #3 scaffolding").
  - **Runner + reporter** ([src/eval/runner.py](../../src/eval/runner.py), [src/eval/report.py](../../src/eval/report.py)) — eval run'larını JSONL + markdown'a yazıyor.
  - **CLI** ([scripts/eval_run.py](../../scripts/eval_run.py)) — system × voice × sentence × metric çapraz çarpımını yürütüyor.

  Boşluklar:
  - **UTMOSv2 wiring eksik** — quality (naturalness) signal yok
  - **SECS (speaker encoder cosine similarity) yok** — voice cloning fidelity ölçülmüyor
  - **Pin path yok** — eval JSON çıktısı `voices.eval_metrics`'e yazılmıyor; "production-pinned voice" tanımı runtime'da görünmez
  - **VoicePublic.eval_metrics expose'u yok** — entegratör kalite bilgisini okuyamıyor

  Sonuç: Whisper-WER tek başına anlaşılırlık ölçüyor; **premium voice tanımı için yetersiz** (robotic ama net sentez WER'i mükemmel verir; voice clone fidelity, naturalness, prosody hiçbiri ölçülmüyor).

## Karar

Üç eksen — **3-metric eval suite** (industry standard: intelligibility + naturalness + clone fidelity) + **pin path** (operator-confirmed) + **public expose**.

### 1. Eval metric suite — 3 modern axis

| Eksen | Metric | Backend | Score yönü |
|---|---|---|---|
| Intelligibility | **Whisper-WER + CER** (mevcut) | OpenAI Whisper large-v3 (`pip install openai-whisper`) | lower |
| Naturalness | **UTMOSv2** | `sarulab-speech/UTMOSv2` (HF + git pkg) | higher (1.0–5.0) |
| Clone fidelity | **SECS** (Speaker Encoder Cosine Similarity) | `microsoft/wavlm-base-plus-sv` (transformers, `WavLMForXVector`) | higher (0.0–1.0) |

Bu üçlü 2024+ TTS literatürünün (XTTS, VALL-E, OpenVoice, F5-TTS) standart objective bench seti. Whisper-only "premature pin" riskini kapatır.

**Frontier / out-of-scope:** LLM-as-judge (Claude/GPT-4o with audio), DNSMOS (audio quality), MOS panel (subjective gold). v1+'da değerlendirilir.

### 2. SECS — speaker encoder cosine similarity

Yeni modül `src/eval/metrics/secs.py`.

- **Backend:** `microsoft/wavlm-base-plus-sv` ([HF](https://huggingface.co/microsoft/wavlm-base-plus-sv)) — Microsoft'un VoxCeleb1 fine-tuned x-vector head'li WavLM modeli; 16 kHz mono input, normalized speaker embedding output, cosine sim ile karşılaştırılır.
- **Reference audio** voice-level (per-sentence değil) — `SECSMetric` construction'da `reference_audio_pcm: bytes` + `sample_rate` alır. CLI (`scripts/eval_pin.py`) voice'un reference audio'sunu R2/local'den okur, metric instance'ını oluşturur, registry'ye stamp'ler.
- **Bu seçim, mevcut `Metric` protocol'ünü değiştirmez** — alternatif (protocol'e `reference_audio_pcm` kwarg ekleme) daha invazifti; runner katmanına dokunmadık.
- **Score range:** 0.0 (alakasız konuşmacı) – 1.0 (aynı kayıt). Tipik thresholds: voice cloning paper'larında >0.6 "convincing clone", >0.75 "high-fidelity clone" pratik kuralı.

### 3. UTMOSv2 wiring (stub → real)

`src/eval/metrics/utmosv2.py` `_load()` `NotImplementedError` → gerçek `utmosv2` paketi çağrısı. Pattern:

```python
import utmosv2
self._model = utmosv2.create_model(pretrained=True)
# ... score():
score = self._model.predict(input=waveform_array, input_sr=16000)
```

Operator opt-in: `pip install git+https://github.com/sarulab-speech/UTMOSv2.git`. Best-practice fallback: `_load()` `ImportError` yakalar, `MetricResult(score=nan, detail={"error": "utmosv2 not installed"})` döner — production-friendly degrade.

### 4. Eval payload schema (eval_metrics JSONB)

Zorunlu key'ler — `voices.eval_metrics` blob bu şekli garanti eder:

```json
{
  "schema_version": 1,
  "evaluated_at": "2026-05-28T14:23:00Z",
  "test_set": {
    "slug": "v0.1-mini",
    "version": 1,
    "sentence_count": 12
  },
  "model": {
    "voxcpm_version": "2.0.x",
    "lora_adapter_uri": "s3://...",
    "lora_adapter_sha256": "...",
    "frontend_pack_id": "tr",
    "lexicon_id": null
  },
  "metrics": {
    "whisper_wer":   {"score": 0.043, "p95": 0.071, "n": 12, "direction": "lower"},
    "whisper_cer":   {"score": 0.011, "p95": 0.018, "n": 12, "direction": "lower"},
    "utmosv2":       {"score": 4.12,  "p95": 3.89,  "n": 12, "direction": "higher"},
    "secs":          {"score": 0.78,  "p95": 0.71,  "n": 12, "direction": "higher"}
  },
  "report_uri": "experiments/2026-05-28-pin-tr-warm-storyteller-v0/REPORT.md",
  "operator_id": "uuid",
  "notes": null
}
```

`schema_version: 1` — gelecek genişlemeler bump'lanır. Eksik metric tolere edilir (örneğin SECS yoksa key yok) — pin yapan operator quality budget'ini bilinçli seçer.

### 5. Pin path — operator-confirmed

**`VoiceRepo.pin_eval(voice_db_id, payload)`** — repo metod:
- Payload `schema_version` + `evaluated_at` + `metrics` zorunlu (app-layer validation)
- `voices.eval_metrics` JSONB üzerine yazılır (replace, append değil — pin = en son geçerli ölçüm)
- Idempotent: aynı payload tekrar verilirse no-op

**`POST /admin/voices/{voice_db_id}/eval-pin`** — operator endpoint:
- JSON body: full payload (yukarıdaki schema)
- `confirm=true` form gerekmiyor; pin idempotent + reversible (operator yanlış pin attıysa yeni run sonucuyla overwrite eder)
- Audit log: `action='voice.eval_pin'`, payload özetini taşır

**`scripts/eval_pin.py`** — end-to-end CLI:
- `--voice-id <slug> --test-set <slug> --metrics wer,cer,utmosv2,secs --output-dir experiments/`
- Eval run'ı çalıştırır (mevcut `eval_run.py` mantığı reuse)
- Çıktı raporundan eval_metrics payload'unu inşa eder
- DB'ye direkt yazar (operator credentials via env) VEYA admin endpoint'e POST eder
- v0'da direct-DB yazıyor; production-ortamda admin endpoint kullanılır

### 6. Promotion gate — şimdi DEĞİL

`release_status='production'` transition'ı `eval_metrics IS NOT NULL` zorlamıyor. Sebep:
- Mevcut voice'ların hiçbirinde eval_metrics yok → migration sırasında tüm production voice'lar invalid olur
- Önce eval pin path canlıya alınmalı, operator önce mevcut voice'ları pin'lemeli, sonra gate eklenmeli

**Follow-up:** gate `VoiceRepo.set_release_status`'a eklenir ayrı bir küçük PR'da (CLAUDE.md ertelenmiş #X).

### 7. Public exposure

`VoicePublic.eval_metrics: dict | None` — entegratör `GET /v1/voices/{id}` response'unda quality bilgisini okur. SDK'lar `if voice.eval_metrics is not None: ...` ile premium voice ayırt edebilir.

Privacy: payload'da `operator_id` var; bu **maskelenmemiş** geçer (operator UUID public-safe; tenant-isolation gerektiren bir alan değil).

### 8. Dependency yönetimi

Yeni `[project.optional-dependencies] eval` grubu:

```toml
[project.optional-dependencies]
eval = [
    "openai-whisper>=20240930",     # Whisper-WER / CER
    "jiwer>=3.0",                    # WER / CER computation
    # transformers >= 4.46 already in main deps — WavLM-base-plus-sv
    # for SECS. utmosv2 is a git package; operator installs separately:
    # pip install git+https://github.com/sarulab-speech/UTMOSv2.git
]
```

UTMOSv2 main deps'e koymuyoruz — git URL pypi pip cache miss'i + git clone her install'da, operatör bir kez kurar.

## Sebep

- **Whisper-WER tek başına = anlaşılırlık only.** Premium voice tanımı için yetersiz; clone fidelity + naturalness ölçülmedikçe "this voice is production-ready" iddiası zayıf.
- **2024+ industry standardı:** WER + UTMOS + SECS üçlüsü XTTS / VALL-E / OpenVoice / F5-TTS paper'larında objective bench. ElevenLabs / MiniMax açık eval yayınlamıyor ama akademik literatür bu üçlüyü kullanıyor.
- **SECS reference audio voice-level:** Construction-time pass, protocol değişmiyor; runner'a dokunmadık. Alternatif (protocol kwarg) daha invazif + her metric'in `**_kwargs` taşıması gerek.
- **UTMOSv2 graceful degrade:** `ImportError` → `score=nan` (NotImplementedError yerine). Operator paketi kurmadıysa pipeline crash etmesin.
- **Pin path admin endpoint + CLI:** Operator iki yoldan pin'leyebilir — UI/manual JSON paste (endpoint) veya end-to-end script (CLI). Production'da endpoint güvenli; CLI workshop ergonomics.
- **Promotion gate ertelendi:** Gate koyarsak mevcut voice'lar invalid olur; pin path canlıya almak + voice'ları pin'lemek + sonra gate doğru sıra.
- **Optional-deps grubu:** Eval ML deps gateway/worker minimum'una yük olmamalı. `pip install neurovoice[eval]` opt-in.
- **Schema_version:** payload blob'u versiyonlu; ilerde 4. metric eklendiğinde v2 bump'lı eski pin'ler valid kalır.

## Risk

- **UTMOSv2 İngilizce-merkezli:** Türkçe için MOS proxy validity sorgulanabilir. Mitigasyon: aynı voice'un kendi-zaman serisinde drift yakalama amacıyla kullanılır (absolute MOS yorumu yerine relative change).
- **WavLM-base-plus-sv VoxCeleb-bias:** İngilizce/multi-lingual genel; non-English voice clone'larında bias olabilir. Mitigasyon: SECS'i da relative metric olarak yorumla (aynı voice'un farklı zamanlardaki SECS'i karşılaştırılır).
- **`utmosv2` git package'ı pypi'de yok:** Operator extra install adımı; CI/Docker setup talimatları runbook'a girer.
- **`eval_metrics` blob şekli versiyonlu ama validation yumuşak:** Pydantic strict-mode değil — bilinmeyen key tolere edilir. Drift riski: schema_version bump'sız eklenen alanlar entegratörü şaşırtabilir.
- **Pin reversibility:** Pin idempotent ama operator yanlış pin atarsa overwrite gerek. Audit log her overwrite'ı kaydeder; rollback için audit log + manuel SQL.
- **R2 reference audio download CLI'da:** `scripts/eval_pin.py` voice'un reference audio'sunu R2'den çekecek (SECS için zorunlu); CLI operator credentials gerektirir.
- **Promotion gate erteleme + production'daki pinsiz voice'lar:** İlk pin canlıya alınana kadar "production voice" tanımı zayıf. Operator runbook bunu kabul eder.

## Alternatifler (reddedildi)

| Seçenek | Niye reddedildi |
| --- | --- |
| WER-only pin (A yolu) | Premature pinning; "premium voice" tanımı yarım, 2 hafta sonra SECS+UTMOS eklemek zorunda kalırız, blob şekli kırılır |
| Protocol'e `reference_audio_pcm` kwarg ekle | Tüm mevcut metric implementations'a `**_kwargs` patch'i; daha invazif, mevcut sözleşmeyi gereksiz değiştiriyor |
| `voice_consent_records` benzeri ayrı `eval_pins` tablo (1:N voice → pin history) | Pin history audit zenginleştirici ama v0 ihtiyacı değil; JSONB blob "en son geçerli pin" semantiği için yeterli; tarihçe `audit_log`'tan okunabilir |
| Promotion gate bu ADR'de | Mevcut voice'lar invalid olur; pin path + bulk pinning önce, gate sonra |
| MOS panel (insan rater) eval'i bu ADR'de | Operasyonel iş akışı, kod değil; out-of-scope |
| DNSMOS (audio quality) bu ADR'de | 4. metric overkill v0'da; ayrı PR |
| Mevcut UTMOSv2 stub'ı NotImplemented bırak | Operator manuel rebuild zorunda kalır; gerçek wiring iki saatlik iş |
| utmosv2 main deps'e koy | Git pkg → her pip install git clone; gateway/worker boot'u yavaşlatır |
| eval_metrics promotion gate'i hemen ekle ama mevcut voice'ları grandfather'la | Special-case logic uzun ömürlü değildir; "sonra gate ekle" daha temiz |
